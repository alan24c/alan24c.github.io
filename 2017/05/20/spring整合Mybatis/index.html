配置 DAO 接口,映射到具体的 xml 文件

  • xml 文件中 < mapper namespace = “…”> < /mapper>标签
    • 通过上述标签映射当前的动态 sql 到 java 代码中的 DAO 接口层
    • 可以认为 xml 文件是 DAO 接口的一个实现类

配置 mapper 接口,使其动态生成 bean

生成 mapper 接口代理

使用 mybatis时，一般定义 mapper 接口,但是 interface 接口类型不能直接在 spring 注册为 bean,使用 @MapperScan 注解来配置 mapper 接口,生成相应的 bean

  • @MapperScan(value=”…”)
    • 使用 value 属性配置 mapper 文件的包
    • 将自动扫描 mapper 接口,生成代理注入到 Spring 中去
将上述配置类生成 bean,加载到 spring 中

  • @Configuration 注解

配置 session 工厂

SqlSessionFactoryBean 实现了 FactoryBean，所以最终创建的 bean 不是 SqlSessionFactoryBean 本身,e而是工厂类下 getObject() 返回的方法的结果.
不需要显示的使用 SqlSessionFactoryBean，相反 getObject() 返回的 session 会被注入到 MapperFactoryBean(即上述通过 @MaperScan 注解生成的 bean) 和实现了 SqlSessionDaoSupport 的 DAO 对象层去。

  • 指定 XML mapper 文件的位置
    • 在 sqlSessionFactory 中通过属性 mapperLocations 来指定
    • 在 MyBatis 配置文件中通过 标签来指定.
配置 session 模板

使用 SqlSessionFactoryBean 来生成 session 模板
通过往 SqlSessionTemplate 中的构造函数中，传入(第一个参数位置)sqlSessionFactory，来生成 sqlSession

使用 session 模板来完数据库操作

  • 隐式使用

    • 当调用 SQL 方法(Mapper DAO 接口层)时,SqlSessionTemplate 会保证使用的 session 是和当前 Spring 事务相关的,此外，它管理 session 的生命周期,包括必要的关闭,提交和回归

  • 显示使用

显示调用 sqlSession
public class UserDaoImpl implements UserDao {

  private SqlSession sqlSession;

  public void setSqlSession(SqlSession sqlSession) {
    this.sqlSession = sqlSession;
  }

  public User getUser(String userId) {
    return (User) sqlSession.selectOne("org.mybatis.spring.sample.mapper.UserMapper.getUser", userId);
  }
}
